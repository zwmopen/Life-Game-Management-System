# 商品图片链接获取与替换指南

## 1. 概述

本文档详细介绍了如何为人生游戏管理系统的商品获取合适的图片链接，并将其替换到项目中。我们推荐使用Unsplash平台的免费商用图片，确保图片质量和版权合规。

## 2. 选择图片平台

### 推荐平台
- **Unsplash**：https://unsplash.com/（最推荐，图片质量高，免费商用）
- **Pexels**：https://www.pexels.com/（类似Unsplash）
- **Pixabay**：https://pixabay.com/（支持图片和矢量图）

### 平台优势
- 免费商用，无需授权
- 图片质量高，分辨率大
- 图片种类丰富，易于找到匹配商品的图片
- 链接稳定可靠

## 3. 获取Unsplash图片链接

### 步骤1：访问Unsplash网站
打开浏览器，访问 [Unsplash官网](https://unsplash.com/)

### 步骤2：搜索图片
在搜索框中输入关键词（建议使用英文，资源更丰富），例如：
- 辣条：`chinese spicy snack`
- 手机：`smartphone`
- 咖啡：`coffee cup`

### 步骤3：选择合适的图片
点击你喜欢的图片，进入图片详情页

### 步骤4：获取原始链接
1. 在图片详情页，点击右上角的「Download free」按钮
2. 在弹出的下拉菜单中，选择「Copy link」选项
3. 此时，原始图片链接已复制到剪贴板

**示例原始链接**：
```
https://images.unsplash.com/photo-1564869521641-a52c75637e91
```

## 4. 添加尺寸和裁剪参数

### 支持的参数
- `w`: 宽度（像素）
- `h`: 高度（像素）
- `fit`: 裁剪方式（`crop`表示裁剪，`max`表示保持比例缩放）

### 添加参数的格式
```
原始链接?w=宽度&h=高度&fit=裁剪方式
```

### 示例转换

**原始链接**：
```
https://images.unsplash.com/photo-1564869521641-a52c75637e91
```

**添加参数后**（适合商品卡片的400x400像素）：
```
https://images.unsplash.com/photo-1564869521641-a52c75637e91?w=400&h=400&fit=crop
```

## 5. 替换项目中的图片链接

### 步骤1：找到商品配置文件
项目的商品配置文件位于：
```
constants/shopCatalog.tsx
```

### 步骤2：找到对应商品
在文件中找到需要替换图片的商品，例如辣条：
```typescript
{ 
  id: 's_food_1', 
  name: '辣条一包', 
  description: '廉价多巴胺 (慎用)', 
  cost: 1, 
  type: 'leisure',
```

### 步骤3：替换图片链接
将 `image` 字段的值替换为新的图片链接

### 步骤4：保存文件
保存修改后的 `shopCatalog.tsx` 文件

## 6. 验证图片链接

### 手动验证
1. 将链接粘贴到浏览器地址栏
2. 按Enter键访问
3. 确认能正常显示图片，且尺寸符合要求

### 脚本验证
使用Node.js脚本批量验证图片链接：

```javascript
// verify_image_links.cjs
const https = require('https');

// 需要验证的图片链接
const imageLinks = [
  'https://images.unsplash.com/photo-1564869521641-a52c75637e91?w=400&h=400&fit=crop',
  // 可以添加更多链接
];

// 验证单个图片链接
function checkImageLink(url) {
  return new Promise((resolve) => {
    https.get(url, (res) => {
      resolve({ url, status: res.statusCode, contentType: res.headers['content-type'] });
    }).on('error', (err) => {
      resolve({ url, status: 'error', error: err.message });
    });
  });
}

// 验证所有图片链接
async function checkAllImageLinks() {
  console.log('正在验证图片链接...');
  const results = [];
  
  for (const link of imageLinks) {
    const result = await checkImageLink(link);
    results.push(result);
    console.log(`${result.status === 200 ? '✅' : '❌'} ${link}`);
    console.log(`   状态: ${result.status}`);
    if (result.contentType) {
      console.log(`   内容类型: ${result.contentType}`);
    }
    if (result.error) {
      console.log(`   错误: ${result.error}`);
    }
    console.log('');
  }
}

checkAllImageLinks();
```

运行脚本：
```bash
node verify_image_links.cjs
```

## 7. 最佳实践

### 图片选择
- 选择**高清图片**（至少1000x1000像素）
- 选择**主体明确**的图片（商品占比适中，背景简洁）
- 选择**符合商品特性**的图片（如食品类图片要有食欲感）
- 注意**图片版权**（确保可免费商用）

### 链接格式
- 统一使用 `?w=400&h=400&fit=crop` 格式的参数
- 避免使用过长或复杂的URL参数
- 定期检查链接，及时更新失效链接

### 性能优化
- 使用合适的图片尺寸（400x400像素适合商品卡片）
- 利用浏览器缓存图片
- 考虑使用CDN加速图片加载

## 8. 常见问题

### Q: 图片链接失效怎么办？
A: 如果发现图片链接失效，重新按照本文档的步骤获取新的图片链接，并替换到项目中。

### Q: 如何批量获取图片链接？
A: 对于大量商品，可以考虑使用Unsplash API批量获取图片，或使用图片搜索工具辅助获取。

### Q: 可以使用其他平台的图片吗？
A: 可以，但需要确保图片可免费商用，且链接稳定可靠。

### Q: 图片尺寸必须是400x400吗？
A: 400x400是推荐尺寸，适合商品卡片的显示。如果需要不同尺寸，可以调整链接中的 `w` 和 `h` 参数。
