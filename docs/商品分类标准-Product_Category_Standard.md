# 商品分类标准

## 1. 概述

本文档定义了人生游戏管理系统中商品的标准分类体系，确保商品数据的统一性和可维护性。

---

## 2. 商品分类体系

### 2.1 主分类列表

| 分类名称 | 说明 | 商品类型 |
|----------|------|----------|
| 数码 | 数码产品、电子设备 | physical |
| 家居 | 家居用品、家具、房产、车辆 | physical |
| 形象设计与穿搭 | 服装、配饰、美容美发 | physical |
| 吃喝 | 食品、饮料、餐饮 | leisure |
| 饮食 | 食品、饮料（备用分类） | leisure |
| 休闲娱乐 | 娱乐活动、票务、旅游 | leisure, rights |
| 会员充值 | 会员服务、充值卡、社群 | rights |

### 2.2 商品类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| `physical` | 实体商品，可持有 | 手机、衣服、家具 |
| `leisure` | 休闲消费，一次性体验 | 餐饮、娱乐活动 |
| `rights` | 权益类商品，具有时效性 | 会员、票务、充值卡 |

---

## 3. 商品ID命名规范

### 3.1 ID格式

```
{类型前缀}_{分类缩写}_{序号}
```

### 3.2 类型前缀

| 前缀 | 商品类型 |
|------|----------|
| `p_` | physical（实体商品） |
| `s_` | leisure（休闲消费） |
| `r_` | rights（权益类） |

### 3.3 分类缩写

| 缩写 | 分类名称 |
|------|----------|
| `dig` | 数码 |
| `gear` | 装备 |
| `home` | 家居 |
| `cloth` | 服装 |
| `food` | 饮食 |
| `ent` | 娱乐 |
| `hair` | 理发 |
| `spa` | 按摩 |
| `books` | 书籍 |
| `tick` | 票务 |
| `vip` | 会员 |
| `char` | 充值 |
| `misc` | 其他 |
| `sport` | 运动 |
| `gift` | 礼物 |
| `property` | 房产 |
| `car` | 车辆 |
| `service` | 服务 |

### 3.4 示例

```
p_dig_1      // 实体商品-数码-1（iPhone 16 Pro）
s_food_1     // 休闲消费-饮食-1（辣条一包）
r_vip_1      // 权益类-会员-1（网易云VIP）
p_home_1     // 实体商品-家居-1（室内炉锅桌子）
p_cloth_1    // 实体商品-服装-1（衣服一件）
```

## 4. 商品数据结构

### 4.1 ShopItem 接口

```typescript
interface ShopItem {
  id: string;           // 商品ID，唯一标识
  name: string;         // 商品名称
  description: string;  // 商品描述
  cost: number;         // 价格（金币）
  type: 'physical' | 'leisure' | 'rights';  // 商品类型
  owned: boolean;       // 是否已拥有
  icon: React.ReactNode; // 图标组件
  category: string;     // 分类名称
  image: string;        // 商品图片URL
}
```

### 4.2 图标规范

- 使用 `lucide-react` 图标库
- 图标尺寸统一为 `size={24}`
- 颜色使用 Tailwind CSS 类名，如 `text-blue-500`

### 4.3 图片规范

- 使用 Unsplash 图片
- 尺寸参数：`w=400&h=400&fit=crop`
- URL格式：`https://images.unsplash.com/photo-{id}?w=400&h=400&fit=crop`

## 6. 添加新商品流程

1. 确定商品分类和类型
2. 按命名规范生成ID
3. 准备商品信息（名称、描述、价格）
4. 选择合适的图标和图片
5. 在 `constants/shopCatalog.tsx` 中添加商品数据
6. 验证商品显示正常

---

## 7. 注意事项

- 商品价格应合理，符合现实逻辑
- 图标颜色应与分类主题协调
- 图片URL确保可访问
- 分类名称保持一致，避免重复定义
