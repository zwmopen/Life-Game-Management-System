## 优化规范

### 套话识别模式

```
❌ "这是一个思维模型，展示了XXX的核心概念和应用。"
❌ "思维模型的核心原则：XXX"
❌ "1. 理解模型的核心概念；2. 结合实际问题应用；3. 定期复盘和优化"
❌ "将XXX应用到日常决策和问题解决中，观察效果并调整。"
❌ "思维模式、决策分析、问题解决"
```

### 正确内容模式

```
✅ deepAnalysis: 解释模型的本质、来源、核心洞察
✅ principle: 3-5条具体原则，每条换行
✅ scope: 具体应用领域，用顿号分隔
✅ tips: 3-5条具体可操作的技巧，每条换行
✅ practice: 3-5条具体实践步骤，每条换行
✅ scenario: 3-5个具体应用案例，每条换行
```

### 换行规范

- JSON 中使用 `\n` 表示换行
- 前端使用 `whitespace-pre-line` 样式显示换行

---

## 优化流程

1. 读取模型数据
2. 检查是否符合规范
3. 替换套话为具体内容
4. 添加换行符
5. 验证 JSON 有效性
6. 保存更新
