# 思维模型图表绘制标准 (V3.0)

## 1. 核心视觉架构 (Layout)
*   **上图下文模式**：屏幕上半部分（50vh）必须为纯净的图表展示区，下半部分为深度文案区。
*   **呼吸感控制**：取消所有显眼的卡片边框和阴影，采用“无限画布”设计理念，文字与图形直接漂浮在背景色上。
*   **分割线标准**：仅允许使用透明度为 `0.06` 的细线进行横向或纵向分割。

## 2. 绘图美学规范 (SVG Design)
*   **动态表现**：SVG 必须包含有意义的动画（如：`animateTransform`、`dasharray` 动效），模拟物理世界的重力、涟漪或流动。
*   **配色方案**：
    *   主色：`var(--accent)` (默认蓝色 #3b82f6)
    *   警示：`#ef4444` (红色)
    *   成功/安全：`#10b981` (绿色)
*   **滤镜应用**：图形应带有淡淡的 `drop-shadow` 或 `blur` 处理，增强视觉的层级感和高级感。
*   **容器比例**：SVG 采用 `viewBox="0 0 800 600"`，确保在 iframe 中水平垂直居中且不被裁切。

## 3. 文案排版标准 (Typography)
*   **大标题 (H1)**：字号 `4rem` 至 `4.5rem`，字重 `900` (Black)，字间距 `-0.05em`。
*   **正文 (P)**：字号 `1.15rem` 至 `1.25rem`，行高 `2.0`。
*   **分栏布局**：详细解析采用 `1:1` 对称网格（Grid），标题使用 `0.8rem` 的极细全大写（或黑体）并配合宽间距。

## 4. 逻辑索引引擎 (Indexing)
*   **绝对倒序**：系统必须维持 165 个节点的绝对倒序关系，序号从 165 降序至 1。
*   **内容锁定**：左侧列表必须展示真实的模型名称，拒绝“占位符”。
