# 商品图片链接修复提示词

## 修复背景
在人生游戏管理系统中，商品图片出现以下问题：
1. 部分商品图片链接无效（404错误）
2. 商品图片与标题不匹配（如辣条显示风景图，烧烤显示不相关图片）
3. 错误使用了picsum.photos等用户拒绝的图片源
4. 所有图片必须使用Unsplash链接

## 修复要求
1. **绝对禁止**使用picsum.photos、via.placeholder.com等非Unsplash图片源
2. 所有商品图片必须**与商品标题高度相关**（如辣条显示辣条图片，烧烤显示烧烤图片）
3. 确保所有图片链接**可正常访问**（无404错误）
4. 修复后必须**验证所有链接有效性**
5. 修复后必须**打开预览验证效果**
6. 遇到浏览器ORB（Origin Restrictions Blocking）机制时，需更换为更可靠的Unsplash链接

## 修复步骤
1. **分析问题**：使用脚本检查所有商品图片链接，记录无效链接和不匹配的图片
2. **编写修复脚本**：
   - 创建图片链接检查脚本，验证所有链接有效性
   - 创建修复脚本，替换无效链接为正确的Unsplash链接
3. **执行修复**：
   - 针对每个无效/不匹配的商品图片，查找**与商品标题相关**的Unsplash图片
   - 使用脚本批量替换或直接编辑文件
4. **验证修复**：
   - 运行图片链接检查脚本，确保所有链接有效
   - 启动开发服务器，打开预览验证效果
   - 逐个检查商品图片是否与标题匹配

## 关键技术点
1. **图片源要求**：仅允许使用Unsplash链接，格式为`https://images.unsplash.com/photo-XXXXXXXXXXXX?w=400&h=400&fit=crop`
2. **商品匹配原则**：
   - 数码产品：显示对应数码设备图片
   - 饮食类：显示对应食物图片
   - 服务类：显示对应服务场景图片
   - 娱乐类：显示对应娱乐活动图片
3. **验证方法**：使用Node.js脚本批量验证图片链接状态码
4. **预览验证**：启动Vite开发服务器，访问`http://localhost:3000/Life-Game-Management-System/`验证效果

## 示例修复流程
1. 运行`node check_image_links.js`检查无效链接
2. 针对无效链接，查找相关Unsplash图片
3. 使用`node fix_image_links_direct.js`直接替换无效链接
4. 再次运行检查脚本验证修复结果
5. 启动开发服务器并打开预览验证

## 注意事项
1. 替换图片链接时，确保**正则表达式精确匹配**，避免误替换
2. 修复后必须进行**双重验证**（脚本检查+预览验证）
3. 遇到浏览器ORB阻止时，更换为不同的Unsplash图片链接
4. 保持代码格式一致性，避免引入语法错误
5. 所有修复操作必须在**开发环境**中验证通过后再部署

## 最终验收标准
1. 所有商品图片链接**均有效**（无404错误）
2. 所有商品图片**与标题高度相关**
3. 仅使用**Unsplash图片源**
4. 开发服务器可**正常启动**
5. 预览页面中**所有商品图片正常显示**
