# 思维模型内容优化规范

## 优化目标

确保每个思维模型都包含完整、具体、有价值的内容，避免套话和空洞表述。

---

## 文本结构标准

每个思维模型必须包含以下6个字段：

| 字段 | 说明 | 要求 |
|------|------|------|
| `deepAnalysis` | 深度分析 | 解释模型的本质、来源、核心洞察 |
| `principle` | 核心原则 | 3-5条具体原则，每条都要有实际含义 |
| `scope` | 适用范围 | 列出具体的应用领域，用顿号分隔 |
| `tips` | 使用技巧 | 3-5条具体可操作的技巧 |
| `practice` | 实践建议 | 3-5条具体的实践步骤 |
| `scenario` | 应用场景 | 3-5个具体的应用案例 |

---

## 内容质量标准

### ❌ 禁止的套话模式

```
"这是一个思维模型，展示了XXX的核心概念和应用。"
"思维模型的核心原则：XXX"
"1. 理解模型的核心概念；2. 结合实际问题应用；3. 定期复盘和优化"
"将XXX应用到日常决策和问题解决中，观察效果并调整。"
"适用于XXX分析、XXX优化等场景。"
```

### ✅ 正确的内容模式

```
deepAnalysis: "XXX是由[提出者]提出的[类型]模型，用于[核心用途]。它的核心洞察是[本质洞察]。这一模型强调[关键特点]，帮助[目标用户]实现[核心价值]。"

principle: "1. [原则名称]：[具体解释]；
2. [原则名称]：[具体解释]；
3. [原则名称]：[具体解释]。"

tips: "1. [具体技巧]：[如何操作]；
2. [具体技巧]：[如何操作]。"

practice: "1. [具体场景]：[具体做法]；
2. [具体场景]：[具体做法]。"

scenario: "1. [场景名称]：[具体应用方式]；
2. [场景名称]：[具体应用方式]。"
```

## 优化示例

### 优化前（套话）

```json
{
  "deepAnalysis": "这是一个思维模型，展示了MVA协议的核心概念和应用。",
  "principle": "思维模型的核心原则：MVA协议",
  "tips": "1. 理解模型的核心概念；2. 结合实际问题应用；3. 定期复盘和优化",
  "practice": "将MVA协议应用到日常决策和问题解决中，观察效果并调整。"
}
```

### 优化后（具体内容）

```json
{
  "deepAnalysis": "MVA协议（Minimum Viable Action，最小可行行动）是一种反完美主义的执行策略。它要求在只有60%准备度时就启动行动，通过真实的市场反馈来指导后续迭代。这一协议的核心洞察是：完美主义本质上是恐惧的避难所，而真实世界的反馈远比闭门造车100天的完美蓝图更有价值。",
  "principle": "1. 60%启动法则：当准备度达到60%时就启动，不等待完美条件；2. 反馈优先：真实市场的负面反馈，价值远超100天的完美构思；3. 行动即学习：通过执行获取认知，而非通过思考预测结果。",
  "tips": "1. 设定60%启动线：明确什么是'足够好'的启动标准；2. 预设反馈机制：在启动前就设计好如何收集用户反馈；3. 接受'烂开始'：第一版可以粗糙，但必须能跑通核心流程。",
  "practice": "1. 项目启动：明确60%准备度的具体标准，如'核心功能可用'；2. 内容发布：完成初稿就发布，通过读者反馈来优化；3. 产品上线：先发布MVP版本，收集用户反馈后再迭代。",
  "scenario": "1. 产品发布：新功能开发完成后，先上线给部分用户试用；2. 内容创作：写博客时，完成初稿就发布；3. 创业启动：有了商业想法，先用最小成本验证。"
}
```

## 排版规范

1. **分点格式**：使用 `1. 2. 3.` 数字编号
2. **分点换行**：每个分点之间用分号 `；` 分隔
3. **重点强调**：关键概念用 `【】` 或加粗标注
4. **具体可操作**：每条建议都要有具体的行动指引

---

## 检查清单

优化完成后，逐项检查：

- [ ] `deepAnalysis` 是否解释了模型的本质和来源？
- [ ] `principle` 是否有3-5条具体原则？
- [ ] `scope` 是否列出了具体应用领域？
- [ ] `tips` 是否有具体可操作的技巧？
- [ ] `practice` 是否有具体的实践步骤？
- [ ] `scenario` 是否有具体的应用案例？
- [ ] 是否避免了所有套话模式？
- [ ] 排版是否清晰、分点明确？

---

## 批量优化流程

1. 读取 `thinkingModels.json` 文件
2. 遍历所有模型，识别套话内容
3. 按照本规范逐个优化
4. 保存更新后的文件
5. 验证优化效果
